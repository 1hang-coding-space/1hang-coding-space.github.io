JAVA

之前惯用语言是python和JS，分别做机器学习和前端，为了深入理解开发过程，正在系统学习JAVA，记录一下学习过程。

Basic

项目结构

项目-（模块）-包-类：可以理解为项目是一个软件；模块是软件下的不同功能分区，负责大的功能；包是为了实现这个功能的不同区域；包下面的类是具体做事情的部分。
资源调用：一个包下面的资源都是共享的：package packageName（除非是private）；不同包下面的资源相互引用时，需要引入包：import packageName.className；不同项目之间的资源不共享
.idea文件：编译的配置文件，编译的内容包括可以控制编译之后的类的位置，工作空间的相关配置，库的配置，编码配置和其他的插件配置等等
out文件：存放JAVA编译后的字节码

编译型与解释型

编译型语言：将源代码编译成可被操作系统直接执行的机器码（如exe文件），一次编译多次运行；但是与特定平台相关，不一定能移植到其他平台。
解释型语言：源代码在运行的过程中被解释器动态翻译并执行，只要其他平台上有对应的解释器就能顺利移植。
JAVA编译特点：JAVA将.java文件编译成.class文件，但这并不是机器可以识别的语言，而是字节码，最终还是需要jvm的解释才能在各个平台执行；不同平台只需要用相同标准实现一份java虚拟机，就可以保证同一套java代码在不同平台都能运行。
如果使用命令行运行JAVA程序，先通过javac把java文件编译成jvm可运行的.class文件：javac className.java，然后在src目录下运行class文件：java packageName.className

强类型/弱类型；动态类型/静态类型

强定义类型语言：不对变量的类型做隐式转换，如C#/Python。5 + ‘5’会报错。
弱定义类型语言：对变量的类型做隐式转换， 如JS/PHP。JS中5 + “5” = “55”, 5 - “5” = 0。

静态类型语言：编译时就确定变量类型，整个生命周期不改变，如JAVA/C/C++，int func(int a){…}。
动态类型语言：运行时才确定变量类型，如Python，def func(a):…，并不知道参数和返回值的类型，只有到运行时调用函数，才最终确定参数和返回值类型。


参考资料：https://c.biancheng.net/view/8803.html

方法

方法是解决一类问题的步骤的有序组合，包含在类或对象中；*System.out.println()*中System是系统类，out是标准输出对象，println()是一个方法。
重载：一个类的两个方法具有相同的函数名称，但有不同的参数列表（包括参数个数、参数类型、参数排列）。 **
可变参数：在参数类型后加一个省略号，接收不定个数的参数；一个方法中只能指定一个可变参数，且它必须是方法的最后一个参数。任何普通的参数必须在它之前声明。

Error & Exception

异常体系结构：JAVA把异常当作对象来处理，并定义一个基类java.lang.Throwable作为所有异常的超类。

检查型异常：编译器要检查这些异常。例如：一个读取文件的方法代码逻辑没有错误，但程序运行时可能会因为文件找不到而抛出FileNotFoundException，如果不处理这些异常，程序将来肯定会出错。所以编译器会提示你要去捕获并处理这种可能发生的异常，不处理（try…catch…）就不能通过编译。
运行时异常：编译器不会检查这些异常，一般可以避免无需try…catch…处理，比如数组下标越界。
错误Error：不是异常，而是脱离程序员控制的问题。由JVM生成并抛出，当JVM不再有继续执行操作所需的内存资源时，将出现OutOfMemoryError，这些异常发生时，JVM一般会选择线程终止。

快捷键：Ctrl + Alt + T

异常捕获：
多层级捕获范围从小到大或互不相交，总之后面的异常范围不能包含前面的异常

自定义异常：
继承Exception类（默认是检查型异常），通过重写toString()方法来自定义异常；

Data Type & Variable

8种基本数据类型

整数数值型（一字节byte，两字节short，四字节int，八字节long L）
小数数值型（四字节float f，八字节double d）
小数默认是double，因此当初始化一个float变量时一定要加f，不然会报错java: 不兼容的类型: 从double转换到float可能会有损失；
float和double有舍入误差，尽量不要比较；如果要比较用BigDecimal

逻辑型（一位boolean）
字符型（两字节char）
用单引号包围的是char，用双引号包围的是String。
字符型变量通过(int)可以转换为数字，即字符的Unicode码值（ASCII码一字节，Unicode码通常两字节，UTF-8是变长的Unicode码）。
定义字符时可以用\u转义，从Unicode码获取字符。

引用类型

类、接口、数组
数组
特点：长度确定不可改变，所有元素是相同类型，元素可以是引用类型。
存储方式：变量名存储在栈内存中，new创建的数组对象存储在堆内存中。数组作参数时传递的是地址，可以直接改变原数组的值，这点和Python一样。

类型转换

强制转化级别由高到低：double -> float -> long -> int -> byte,short,char
从高到低可以直接转化（如果数值到字符就生成Unicode码），从低到高需要强制转化。

数组类型不能自动低转高；例如定义一个形参为double的函数，可以在主方法中输入int进行调用；但如果定义一个形参为double[]的函数，不能输入int[]进行调用。

判别符

==如果比较的是8种基本数据类型，直接比较值；如果比较的是引用类型，比较内存地址。

包装类

基本类型的变量不是对象，不能通过obj.getClass()方法直接获取其Class，需要用到包装类（Byte、Short、Integer、Long、Float、Double、Boolean、Character）；
包装类中的SIZE、MIN_VALUE、MAX_VALUE等静态常量记录了相关信息，例如Integer.MAX_VALUE = 2^31 - 1 = 2147483647，SIZE的单位是bit。

Object Oriented

面向对象编程的本质：以类的方式组织代码，以对象的方式封装数据。

方法

静态方法：用static修饰符修饰。不需要实例化这个类，如果实例化再调用不会报错，但会提示“不应该通过类实例访问静态成员”；
在同一个包下public的静态方法可以直接通过类.方法调用（不同的包下则需要import packageName.className）；
如果在同一个类中，主函数可以直接调用主函数外的静态方法，不必要加类名，当然加了也不会报错。
非静态方法：没有用static修饰符修饰，需要new实例化这个类后，通过对象调用。
原理：静态方法是和类一起加载的，非静态方法是类实例化之后才存在，因此不能在静态方法中调用非静态成员。

构造方法

构造器/构造方法：方法名必须和类的名称相同，必须没有返回值但也不能写void。
构造函数类似于Python中的__init__，在把类实例化时调用，分为有参构造和无参构造，构造方法只能包含初始化代码，不能包含方法定义。
使用new关键字，本质是调用构造器。
一旦定义了有参构造，如果还想使用无参构造，则无参构造必须显式定义，具体调用哪个取决于输入的参数，毕竟方法可以重载。

封装

程序设计追求高内聚，低耦合，高内聚就是类的内部数据操作细节自己完成，不允许外部干涉；低耦合就是仅暴露少量的方法供外部使用。
封装通常是数据的隐藏，禁止直接访问一个对象中数据的实际表示，而是通过操作接口来访问。
封装的意义：提高程序的安全性，保护数据；隐藏代码的实现细节；统一接口；提高系统的可维护性。

访问修饰符
封装的核心：属性私有，get/set，插入get/set方法的快捷键：Alt + Insert
public：允许其他所有类访问该成员，无论是在包内还是包外，包外的话需要import packageName.className。
private：只能在它所属的类内部访问，因此如果要访问、修改的话需要在类内定义一些外部方法get、set。私有的属性或方法无法被子类继承。
protected：只能让子类访问该成员。
default：包级别访问（Package-private），成员只能在同一个包内被访问，不能在其他包中被访问，即使import。

继承

一个类继承另一个类的属性和方法。在JAVA中所有的类都直接或间接继承Object类。
JAVA中只有单继承，没有多继承（一个子类只能有一个直接父类）。
super用于在子类中访问父类的属性或方法。
子类的所有构造函数默认调用父类的无参构造函数，即使构造子类时使用的是有参构造并且父类也有有参构造函数，并且父类的构造函数先执行。

方法重写
重写：需要有继承关系，子类重写父类的方法（而非属性）
1.方法名必须相同
2.参数列表必须相同
3.返回类型必须相同
4.修饰符：范围可以相等或扩大但不能缩小 public > protected > default > private； private方法不能被重写
5.抛出的异常：范围可以被缩小但不能扩大

多态

多态：同一方法可以根据发送对象的不同而采用多种不同的行为方式。
一个对象的实际类型是确定的（new了什么就是什么），但可以指向对象的引用类型不确定（父类的引用变量可以指向子类对象）。
定义格式：父类类型 变量名 = new 子类类型();

多态存在的条件：
1.有继承关系
2.子类重写父类方法
3.父类引用指向子类对象

抽象类

用于提供一个模板，提高开发效率。

接口

普通类：只有具体实现
抽象类：具体实现和规范（抽象方法）都有
接口：只有规范约束，从而实现了约束和实现分离

类可以通过implements实现接口
抽象类有局限，不能多继承；但接口没限制
要实现接口的类，必须重写接口中的所有方法

接口就是规范，定义的是一组规则，让不同的类实现；接口的本质是契约，制定好后必须遵守。
面向对象的精髓就是对对象的抽象，最能体现这一点的就是接口。为什么我们讨论设计模式都只针对具备了抽象能力的语言（比如C++/JAVA/C#），就是因为设计模式研究的是如何合理地去抽象。
接口中的所有定义的方法都是public abstract，定义的属性都是public static final。
接口没有构造函数。
