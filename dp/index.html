环绕字符串中唯一子字符串

LeetCode467，链接：https://leetcode.cn/problems/unique-substrings-in-wraparound-string/

把字符串 s 看作 “abcdefghijklmnopqrstuvwxyz” 的无限环绕字符串，所以 s 看起来是这样的：
“…zabcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyzabcd….” 。
现在给定另一个字符串 p 。返回 s 中不同的 p 的非空子串的数量 。 

输入：p = “zab”
输出：6
解释：在字符串 s 中有 p 的六个子串 (“z”, “a”, “b”, “za”, “ab”, “zab”) 。

解：若用蛮力解法，时间复杂度至少为O(n^2)，即对输入字符串的起点和终点都遍历一遍，还需要检查子串是否已被统计过；
采用动态规划的思想，用一个长度为26的数组记录以每个英文单词结尾的子串的最大长度，最后返回数组和，时间复杂度O(n)。

最小移动次数使数组元素相等

LeetCode462，链接：https://leetcode.cn/problems/minimum-moves-to-equal-array-elements-ii/

给定一个长度为 n 的整数数组 nums ，返回使所有数组元素相等需要的最少移动数。
在一步操作中，你可以使数组中的一个元素加 1 或者减 1 。

输入：nums = [1,2,3]
输出：2
解释：
只需要两步操作（每步操作指南使一个元素加 1 或减 1）：
[1,2,3] => [2,2,3] => [2,2,2]

解：目标数是数组的中位数。若对数组排序，从左右两端开始依次取上下界，当且仅当目标数落在范围满足“最小移动次数”，不断逼近直至中位数（若数组长度为奇数）或中间两个数之间任意数（若数组长度为偶数）

tips：js的sort()函数如果不传参，默认为ASCII码非降序排序，若遇到多位数排序很可能无意义；上述代码使用nums.sort((a,b) => a - b)，第一个参数是数组后一个数，第二个参数是数组前一个数，返回值是负数则交换顺序。因此nums.sort((a,b) => a - b)代表非降序排序。

最长递增子序列

LeetCode300，链接：https://leetcode.cn/problems/longest-increasing-subsequence/

给出一个整数数组 nums ，找到其中最长严格递增子序列的长度。
子序列是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

解：创建一个数组list，用于记录数组中以该元素结尾（必须取到改元素）的最长子序列。具体实现方法是依次遍历每个元素i，然后遍历其之前的元素j，若i大于j，则可能实现的最长递增子序列为list[j] + 1，动态更新记录最大值。

粉刷房子

LeetCode 剑指II 091，链接：https://leetcode.cn/problems/JEj789/

假如有一排房子，共 n 个，每个房子可以被粉刷成红色、蓝色或者绿色这三种颜色中的一种，你需要粉刷所有的房子并且使其相邻的两个房子颜色不能相同。
当然，因为市场上不同颜色油漆的价格不同，所以房子粉刷成不同颜色的花费成本也是不同的。每个房子粉刷成不同颜色的花费是以一个 n x 3 的正整数矩阵 costs 来表示的。
例如，costs[0][0] 表示第 0 号房子粉刷成红色的成本花费；costs[1][2] 表示第 1 号房子粉刷成绿色的花费，以此类推。
请计算出粉刷完所有房子最少的花费成本。

输入: costs = [[17,2,17],[16,16,5],[14,3,19]]
输出: 10
解释: 将 0 号房子粉刷成蓝色，1 号房子粉刷成绿色，2 号房子粉刷成蓝色。
  最少花费: 2 + 5 + 3 = 10。

解：用一个长度为3、动态更新的数组，记录从第0号到第i号房子刷三种颜色各自的最小总花费。

无重复字符的最长子串

LeetCode3，链接：https://leetcode.cn/problems/longest-substring-without-repeating-characters/

解：若用蛮力解法，时间复杂度为O(n^2)，考虑到随着起始位置的递增，最长无重复字串的结束位置也必然递增，因此采用滑动窗口/双指针思想，将时间复杂度降为O(n)。同样的思想可用于“最长连续递增序列”等问题中。
