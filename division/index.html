Joseph-ring

链接：https://leetcode.cn/problems/find-the-winner-of-the-circular-game/

n名成员编号为1至n顺时针排成一个圈，顺时针开始1至k报数，剔除报数为k者，求最后的幸存者。

解：若采用蛮力的思想，则构造队列，通过循环将1至(k-1)者先出队列再入队列，将k者出队列，直至队列长度为1

可通过减治思想简化问题，递推式为f(n, k) = (f(n - 1, k) + k) % n, f(1) = 1。

运用递推归纳后，时间复杂度由O(nk)降为O(n)，空间复杂度由O(n)降为O(1)。

construct-smallest-number-from-IDIDID

LeetCode2375，链接：https://leetcode.cn/problems/construct-smallest-number-from-di-string/description/

给你下标从 0 开始、长度为 n 的字符串 pattern ，它包含两种字符，I 表示 上升 ，D 表示 下降 。
你需要构造一个下标从 0 开始长度为 n + 1 的字符串，且它要满足以下条件：
num 包含数字 ‘1’ 到 ‘9’ ，其中每个数字 至多 使用一次。
如果 pattern[i] == “I” ，那么 num[i] < num[i + 1] 。
如果 pattern[i] == “D” ，那么 num[i] > num[i + 1] 。

解：采用分治思想，考虑连续重复的字符串。
比如 III 对应的结果就是1234，后面再出现 DD ，只需把1234的最后一位4消去，再补上654，变成123654，以此类推。
构建一个集合记录已经用掉的数字，不断采集连续重复字符串输出尽可能小的结果，添加到最终结果尾部（需要考虑之前结果最后一位的去除，到最终结尾处再补上）

my-calendar-i

LeetCode729，链接：https://leetcode.cn/problems/my-calendar-i/description/

实现一个 MyCalendar 类来存放日程安排。如果要添加的日程安排不会造成重复预订，则可以存储这个新的日程安排。
当两个日程安排有一些时间上的交叉时（例如两个日程安排都在同一时间内），就会产生重复预订。
日程可以用一对整数 startTime 和 endTime 表示，这里的时间是半开区间，即【startTime, endTime）。

解：用保持元素排序的TreeSet构建，对于每次查询，查找大于等于end的第一个起点l1，然后取其左边的区间的终点r2，若r2 <= start < end <= l1，则该区间可以预定
